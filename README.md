# Windows-Server-Security-Hardening
## Security Controls Implemented

- Disabled Guest account
- Disabled default Administrator account
- Created secure named admin account
- Restricted RDP access
- Configured account lockout policy
- Disabled unnecessary services

 Objective:

To reduce attack surface and protect privileged accounts by applying Windows Server 2022 hardening best practices.

## Security Impact

This configuration helps prevent:

- Brute force attacks
- Unauthorized admin access
- Privilege escalation
- Lateral movement

## Role Alignment

This project is aligned with:
- CyberArk Administrator
- Privileged Access Management (PAM)
- Windows Security Hardening
